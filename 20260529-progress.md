## What I built
- LinkedIn PM Job Scraper — end to end workflow in n8n
- Manual Trigger → HTTP Request (Apify) → Remove Duplicates → Edit Fields → Aggregate → Gmail

## Apify setup
- Chose actor: `supreme_coder/linkedin-post` ($1/1000 posts, no cookies, 4.8 stars)
- 3 search URLs: hiring PM in Gurugram, Bangalore, Hyderabad
- Limit: 10 posts per city, max cost cap $0.10 per run
- Returns: urn, text, url, authorName, authorHeadline, timeSincePosted, numLikes

## n8n workflow details
- HTTP Request: fetches last Apify run results via API
- Remove Duplicates: deduplicates by `urn` (LinkedIn's unique post ID)
- Edit Fields: keeps only 6 fields, cleans URL with `.split('?')[0]`
- Aggregate: combines all posts into one item for single email
- Gmail: sends HTML digest

## What I learned
- How to evaluate and pick an Apify actor (stars, users, last updated, pricing)
- HTTP Request node to call external APIs
- Deduplication by specific field (urn vs url — why urn is better)
- URL cleaning with JavaScript in n8n
- Google Cloud OAuth setup (project → enable API → consent screen → credentials)
- HTML email formatting using `.map()` and `.join()` in n8n expressions

## Known limitations / next steps
