### What I Built
- Extended the LinkedIn PM Job Scraper workflow to add an AI scoring layer.
- Instead of receiving all 30 raw posts, the workflow now reads my resume, scores each post for relevance, and emails only the relevant ones.
- Manual Trigger → HTTP Request (Fetch Resume from GitHub) → Extract from File (PDF → text) → HTTP Request (Fetch Apify Results) → Remove Duplicates → Edit Fields (clean URL, keep 6 fields) → Basic LLM Chain (AI scores each post against your resume) → Code Node (robust JSON parsing with 3 fallback strategies) → Edit Fields (combine score + original fields) → IF Node (score ≥ 7 passes through) → Aggregate → Gmai

### New Nodes Added
- HTTP Request — fetches resume PDF from GitHub (raw URL)
- Extract from File — converts PDF binary to plain text
- Basic LLM Chain — scores each post against resume using Ollama (llama3)
- Code Node (JS) — parses LLM JSON output with 3 fallback strategies to handle
  inconsistent Ollama formatting (extra text, backticks, malformed JSON)
- Edit Fields — combines score/reason with original post fields
- IF Node — filters posts with score ≥ 7 (Number conversion needed since
  score comes through as string)

### What I Learned
- GitHub raw URL vs blob URL — only raw works for n8n HTTP requests
- "Run Once for All Items" mode required when using $input.all() in Code node
- LLM output is inconsistent — need robust parsing, not just JSON.parse()
- Basic LLM Chain drops previous fields — need Edit Fields after to restore them
- Structured Output Parser is too heavy for llama3 — causes empty response errors
- IF node needs Number() conversion when comparing string values to numbers

### Result
28 posts scraped → 14 passed the relevance filter → delivered to Gmail
