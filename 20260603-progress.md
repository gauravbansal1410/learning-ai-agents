## What I Built

### Session 1 — AI Scoring Layer on LinkedIn Job Scraper
Extended the LinkedIn PM Job Scraper workflow to add an AI scoring layer. Instead of receiving all 30 raw posts, the workflow now reads my resume, scores each post for relevance, and emails only the relevant ones.

**Full workflow:**
Manual Trigger → HTTP Request (Fetch Resume from GitHub) → Extract from File (PDF → text) → HTTP Request (Fetch Apify Results) → Remove Duplicates → Edit Fields (clean URL, keep 6 fields) → Basic LLM Chain (AI scores each post against resume) → Code Node (robust JSON parsing with 3 fallback strategies) → Edit Fields (combine score + original fields) → IF Node (score ≥ 7 passes through) → Aggregate → Gmail

**New nodes added:**
- HTTP Request — fetches resume PDF from GitHub (raw URL)
- Extract from File — converts PDF binary to plain text
- Basic LLM Chain — scores each post against resume using Ollama (llama3)
- Code Node (JS) — parses LLM JSON output with 3 fallback strategies to handle inconsistent Ollama formatting (extra text, backticks, malformed JSON)
- Edit Fields — combines score/reason with original post fields
- IF Node — filters posts with score ≥ 7 (Number conversion needed since score comes as string)

**What I learned:**
- GitHub raw URL vs blob URL — only raw works for n8n HTTP requests
- "Run Once for All Items" mode required when using $input.all() in Code node
- LLM output is inconsistent — need robust parsing, not just JSON.parse()
- Basic LLM Chain drops previous fields — need Edit Fields after to restore them
- Structured Output Parser is too heavy for llama3 — causes empty response errors
- IF node needs Number() conversion when comparing string values to numbers

**Result:** 28 posts scraped → 14 passed the relevance filter → delivered to Gmail

### Session 2 — Apify Auto-Trigger (no more manual runs)
Replaced the manual Apify step with a single HTTP Request node that starts the scrape, waits for completion, and returns results — all in one call.

**What changed:**
- Replaced the old "Fetch Apify Results" node with a new HTTP Request node using Apify's `run-sync-get-dataset-items` endpoint
- This single node now does everything: triggers the Apify actor, waits for it to finish, and returns the scraped data
- No manual Apify run needed before triggering the n8n workflow

**Endpoint used:**
```
POST https://api.apify.com/v2/actors/supreme_coder~linkedin-post/run-sync-get-dataset-items?token=***
```

**Request body passed:**
```json
{
    "deepScrape": true,
    "limitPerSource": 10,
    "rawData": false,
    "urls": [
        "https://www.linkedin.com/YOUR_DESIRED_SCRAPPER_URL"
    ]
}
```

**What I learned:**
- `run-sync-get-dataset-items` is cleaner than `waitForFinish` — one node does trigger + wait + fetch
- Empty `{}` body doesn't work — actor input (urls, settings) must be passed explicitly in the body
- Apify doesn't carry over saved UI settings when called via API
- Webhooks exist but are overkill for local n8n — `run-sync` is the right approach here

**Result:** Full workflow runs end-to-end with one click. Apify triggers automatically, scrapes, scores, filters, and emails.
