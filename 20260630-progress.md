## Goal
Finish HTTPS setup on the Oracle Cloud VM so the PM Interview Bot can be hosted on GitHub Pages without browser mixed-content blocks (n8n webhooks were HTTP-only).

## What got done

**Evaluated alternatives before committing**
- Considered Cloudflare Tunnel, n8n Cloud, Railway/Fly.io hosting, and n8n's native SSL env vars before picking nginx + Let's Encrypt.
- Chose nginx: free, permanent, and reusable across all future projects on the same VM (one domain, multiple location blocks).

**Phase 1 — Opened ports 80 and 443**
- Installed and configured UFW (wasn't installed yet). Allowed 22, 80, 443, 5678 before enabling — avoided locking myself out.
- Added ingress rules for ports 80 and 443 in Oracle Cloud Console (Default Security List) — separate from UFW, both layers needed updating.

**Phase 2 — Installed nginx**
- `sudo apt install nginx`, started and enabled the service.
- Hit a wall: nginx was running but site wasn't reachable on port 80.

**Discovered Oracle's hidden third firewall layer**
- `iptables -L INPUT` showed a `REJECT all` rule at line 5, sitting *before* UFW's own chain — UFW rules were never being reached.
- Fixed by inserting explicit `ACCEPT` rules for ports 80 and 443 at position 5 (before the REJECT), then persisted with `iptables-persistent` so it survives reboots.
- Key learning: Oracle Cloud Ubuntu images have **three** firewall layers — UFW, Oracle Cloud Security List, and a default iptables REJECT rule. All three need separate configuration.

**Phase 3 — SSL certificate**
- Installed Certbot, ran `certbot --nginx -d gaurav-n8n.duckdns.org`.
- Cert issued and auto-deployed cleanly, auto-renewal scheduled. Expires 2026-09-28.

**Phase 4 — nginx reverse proxy config**
- Replaced the default nginx site config with a proper reverse proxy block: port 80 redirects to 443, port 443 terminates SSL and proxies to `localhost:5678` (n8n).
- Verified with `nginx -t` before reload each time.

**Phase 5 — n8n environment update**
- Updated `~/n8n.env`: `N8N_PROTOCOL=https`, `WEBHOOK_URL=https://gaurav-n8n.duckdns.org/` (no port number now — nginx handles 443 externally, 5678 is internal-only).
- Full container recreate required (`docker stop` → `docker rm` → `docker run`)
— a simple restart doesn't pick up env file changes.
- Confirmed in n8n UI: webhook Production URL now shows `https://gaurav-n8n.duckdns.org/webhook/pm-interview-question` — clean, no
  port, HTTPS.

**Phase 6 — GitHub Pages + end-to-end test**
- Updated `interview_bot.html` CONFIG object with the new HTTPS Production webhook URLs.
- Enabled GitHub Pages on `learning-ai-agents` repo (source: `main` branch, root folder).
- Live at: `https://gauravbansal1410.github.io/learning-ai-agents/pm-interview-bot/interview_bot.html`
- Full loop tested end-to-end on phone: question generation, model answer, debrief submission, and debrief email all worked over HTTPS.

## Decisions made

## Infrastructure now reusable
nginx + Let's Encrypt on the Oracle VM is no longer single-purpose — any future project (insurance RAG, company research tool, etc.) can get its own clean HTTPS path on the same domain by adding a new `location` block in the nginx config. No repeat of Phases 1–3 needed for future projects.
